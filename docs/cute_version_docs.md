## 🎨 颜色方案

### 主要颜色
- **绿色** (`\x1b[32m`): 蛇身、游戏进行中状态
- **红色** (`\x1b[31m`): 游戏结束状态
- **黄色** (`\x1b[33m`): 分数显示、暂停状态
- **蓝色** (`\x1b[34m`): 速度显示
- **洋红色** (`\x1b[35m`): 最高分显示
- **青色** (`\x1b[36m`): 标题和提示信息

### 颜色使用原则
1. **一致性**: 相同类型的元素使用相同颜色
2. **对比度**: 确保文字在不同背景下清晰可见
3. **美观性**: 选择协调的颜色组合
4. **功能性**: 颜色传达信息（如红色表示危险）

## 🔧 技术实现

### 跨平台兼容性
```c
// 平台检测
#if defined(_WIN32) || defined(WIN32) || defined(__WIN32__) || defined(__NT__)
    #define IS_WINDOWS 1
#else
    #define IS_WINDOWS 0
#endif

// 跨平台清屏
fn clear_screen() {
    #if IS_WINDOWS
        io.print("\x1b[2J\x1b[H");  // Windows 10+ 支持ANSI
    #else
        io.print("\x1b[2J\x1b[H");  // Unix/Linux/macOS
    #endif
}
```

### 表情符号支持
- **UTF-8编码**: 所有表情符号使用UTF-8编码
- **终端要求**: 需要支持UTF-8的终端
- **后备方案**: 提供ASCII后备字符

### 动态界面更新
```c
// 在指定位置绘制字符
fn draw_at(x: i32, y: i32, ch: string) {
    io.print("\x1b[{};{}H", y + 3, x * 2 + 3);
    io.print("{}", ch);
}
```

## 🚀 功能特性

### 1. 完整的游戏系统
- **分数系统**: 每吃一个苹果得10分
- **等级系统**: 每50分升一级，游戏速度加快
- **最高分记录**: 自动保存和显示最高分
- **游戏状态**: 运行中、暂停、结束三种状态

### 2. 用户友好的界面
- **实时信息**: 显示当前分数、等级、速度
- **控制说明**: 清晰的按键说明
- **状态提示**: 游戏状态和操作提示
- **帮助系统**: 按H键显示完整帮助

### 3. 响应式设计
- **动态速度**: 随着等级提高，游戏速度加快
- **智能提示**: 根据分数显示不同的鼓励信息
- **错误处理**: 友好的错误提示和恢复机制

## 🛠️ 构建和运行

### 使用构建脚本
```bash
# 给予执行权限
chmod +x build_cute_version.sh

# 运行构建脚本
./build_cute_version.sh
```

### 手动构建
```bash
# 使用 Zen-C 编译器
zc build -o build/snake_cute src/main_cute.zc

# 或使用 C 编译器 (后备方案)
gcc -o build/snake_cute build/snake_cute.c -lm
```

### 运行游戏
```bash
# 运行可爱版本
./build/snake_cute
```

## 🌍 跨平台支持

### Windows
- **推荐终端**: Windows Terminal (从Microsoft Store安装)
- **要求**: Windows 10+，支持UTF-8和ANSI转义序列
- **备用方案**: PowerShell, Git Bash

### Linux/macOS
- **推荐终端**: 任何现代终端模拟器
- **要求**: 支持UTF-8和ANSI颜色
- **测试通过**: Ubuntu, Fedora, Arch, macOS Terminal

### 终端要求
1. **UTF-8支持**: 能够显示表情符号
2. **ANSI颜色**: 支持颜色转义序列
3. **足够大小**: 至少80x25字符
4. **真彩色支持** (可选): 更好的颜色显示
